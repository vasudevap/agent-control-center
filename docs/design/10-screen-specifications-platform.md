# 6. Approvals

## Purpose

Provide a governed workspace for reviewing and authorizing pending actions.

## Primary Question

> What requires my authorization?

## Layout

- Queue
- Risk summary
- Approval detail
- Evidence panel
- Decision history

## Primary Actions

- Approve
- Reject
- Request More Information

---

# 7. Alerts

## Primary Question

## Layout

- Alert list

- Resolution timeline

## Primary Actions

- Investigate

# Global Screen Behaviors

Every production screen should define:

- Loading state
- Empty state
- Error state
- Success feedback
- Keyboard support
- Responsive behavior
- Accessibility requirements

---

# Navigation Standards

Every detail page should support:

- Breadcrumbs
- Previous / Next navigation
- Related object links
- Deep linking
- URL persistence

---

# Operational States

Every object should consistently represent:

- Healthy
- Running
- Scheduled
- Pending
- Warning
- Failed
- Disabled
- Archived

---

# Acceptance Criteria

Every screen must:

- answer one primary operational question
- preserve context
- expose relationships
- minimize cognitive load
- support investigation before action
- align with Atlas Design Principles
