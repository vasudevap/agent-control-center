# Atlas Visual Identity

**Status:** Draft  
**Version:** 1.0  
**Owner:** Product Design  
**Depends on:** 00-brand.md, 01-design-principles.md

---

# Purpose

This document defines the visual identity of Atlas.

It establishes how Atlas should be perceived through typography, color, iconography, imagery, motion, and brand expression.

It intentionally does not define implementation details or UI components.

---

# Brand Expression

Atlas should communicate:

- Control
- Trust
- Precision
- Clarity
- Reliability
- Governance
- Professionalism

The visual identity should feel timeless rather than trend-driven.

---

# Design References

Atlas should aspire to the product quality of:

- Linear
- Stripe
- GitHub
- Vercel
- Apple
- OpenAI

These references inform quality, not appearance.

---

# Visual Personality

Atlas should feel:

- Calm
- Confident
- Technical
- Structured
- Refined
- Purposeful

Atlas should not feel:

- Futuristic
- Playful
- Experimental
- Consumer-focused
- Decorative

---

# Logo Direction

The logo should communicate:

- orientation
- control
- navigation
- structure
- systems
- precision

Avoid:

- robots
- brains
- sparkles
- chat bubbles
- generic AI symbols
- circuit-board imagery

Potential directions:

- monogram
- compass
- coordinate marker
- control point
- abstract atlas grid
- geometric navigation motif

---

# Wordmark

The wordmark should be:

- clean
- highly legible
- timeless
- balanced
- recognizable at small sizes

Prefer geometric or neo-grotesque typefaces.

---

# Icon Philosophy

Icons should:

- use a consistent stroke system
- communicate operational meaning
- remain recognizable at 16px
- avoid unnecessary detail

---

# Typography

Typography should communicate hierarchy before decoration.

Primary goals:

- excellent readability
- compact information density
- strong numeric alignment
- clear table presentation

Use typography—not color—to establish hierarchy whenever possible.

---

# Color Philosophy

Color communicates state.

Primary palette should remain neutral.

Accent colors should indicate:

- Action
- Success
- Warning
- Critical
- Information

Color should never be decorative.

---

# Spacing Philosophy

Whitespace creates structure.

Spacing should:

- group related information
- separate operational domains
- reduce cognitive load

Generous spacing does not imply sparse information.

---

# Elevation

Use minimal elevation.

Prefer hierarchy through:

- layout
- spacing
- typography
- borders

Avoid excessive shadows.

---

# Borders

Use subtle borders to organize information.

Borders should separate, not dominate.

---

# Motion

Motion should communicate:

- state changes
- navigation
- hierarchy
- progress

Animation should be:

- subtle
- fast
- interruptible

Avoid decorative motion.

---

# Imagery

Atlas should minimize illustrative content.

When imagery is used, prefer:

- abstract geometry
- topology
- coordinates
- networks
- structured systems

Avoid stock photography.

---

# Illustration

Illustrations should be rare.

Operational interfaces should rely primarily on data visualization.

---

# Data Visualization

Charts should emphasize:

- operational health
- trends
- capacity
- reliability
- execution history

Avoid unnecessary visual embellishment.

---

# Dark and Light Modes

Both themes are first-class experiences.

Neither should appear secondary.

Semantic colors must remain consistent across themes.

---

# Accessibility

Visual identity must support WCAG 2.2 AA.

Contrast, typography, focus states, and color usage must satisfy accessibility requirements.

---

# Success Criteria

A first-time viewer should think:

- This feels like professional operational software.
- I trust this platform.
- It appears engineered rather than marketed.
- It is modern without chasing trends.
- It could be used by an enterprise organization.

The visual identity should reinforce Atlas as the unified control plane for an AI workforce.
